The Project:

This app written in Rust is designed to grab a given X users feed and send it as a text. It uses a developer account and API to request posts. It uses SMTP and a Gmail account to send an email to a phone number via the Verizon network ultimately appearing as a text. It passes time parameters to the API by reading a file that holds the most recent date and time of the last request and the current system time. With these values, anyone running the program can get just the feed of a user since when they last checked. Ideally, this can be scheduled to run as a background process a few times a day. There are a few limitations and inconveniences, namely a character limit for texting which requires breaking the message up. Sending individual messages can come in in the wrong order. There also seems to be a limit to the number of characters the API itself will return. Finally, there will be issues with any sort of media. It’s probably better to just open up the X app. But the easy, convenient, and visually pleasing way is too boring.

I got an X account when it was still Twitter to pretend to care about a few public figures. I forgot about it until recently when its primary purpose became following the posts of the one person I care to hear from. Unfortunately, armed with my weak mental fortitude, I fell victim to doom scrolling on X for the first time. Man, this place is crazy. Someone even said something that I don't agree with! I wrote this code so I wouldn’t even be tempted to dive into the moshpit. One day, I would like to have this as a scheduled process on a personal Linux server, but for now, it’s just fun Rust practice and an introduction to interacting with other services. It was also frustrating debugging, since requests are limited to 1 every 15 minutes capping out at 100 per month. I wasted hours wondering why my error messages changed when my code didn’t.

-February 2025
